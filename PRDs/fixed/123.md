这两个缺失点，其实不是“交易员经验”的问题，而是**数据工程问题**。你可以这样理解：

你现在的日报已经能看“今天谁强”；
 缺失的两个东西，是为了判断：

1. **资金是不是连续流入某个板块**；
2. **某只股票是不是属于当前强势板块**。

这两个补上以后，日报才会从“排行榜”变成“有逻辑的复盘系统”。你上传的 `SKILL.md` 里已经提到项目会查询“板块资金占比异动”，并且数据库里有 `board_amount_ratio` 这类表的查询示例，所以方向是对的，只是还需要把数据沉淀和映射补完整。

## 4. 怎么计算 3 日、5 日变化？

有了历史表以后，就可以查：

```

在 Python 里做透视表：

```
def calc_board_ratio_change(board_type="行业", window=3):
    conn = psycopg2.connect(DATABASE_DSN)

    sql = """
    SELECT trade_date, board_type, board_name, amount_ratio, pct_chg, amount
    FROM board_amount_ratio
    WHERE board_type = %s
    ORDER BY trade_date ASC
    """

    df = pd.read_sql(sql, conn, params=(board_type,))
    conn.close()

    df["trade_date"] = pd.to_datetime(df["trade_date"])
    df["amount_ratio"] = pd.to_numeric(df["amount_ratio"], errors="coerce")

    latest_date = df["trade_date"].max()
    latest_df = df[df["trade_date"] == latest_date].copy()

    pivot = df.pivot_table(
        index="board_name",
        columns="trade_date",
        values="amount_ratio",
        aggfunc="first"
    )

    recent_dates = sorted(pivot.columns)[-window:]

    latest_df["ratio_today"] = latest_df["board_name"].map(pivot[recent_dates[-1]])
    latest_df["ratio_before"] = latest_df["board_name"].map(pivot[recent_dates[0]])
    latest_df[f"ratio_change_{window}d"] = latest_df["ratio_today"] - latest_df["ratio_before"]

    latest_df = latest_df.sort_values(f"ratio_change_{window}d", ascending=False)

    return latest_df
```

def get_hot_boards(board_type=None, top_n=20):
    conn = psycopg2.connect(DATABASE_DSN)

    sql = """
    SELECT board_type, board_name, pct_chg, amount_ratio
    FROM board_amount_ratio
    WHERE trade_date = (SELECT MAX(trade_date) FROM board_amount_ratio)
    """

    if board_type:
        sql += " AND board_type = %s"
        df = pd.read_sql(sql, conn, params=(board_type,))
    else:
        df = pd.read_sql(sql, conn)

    conn.close()

    df["score"] = df["pct_chg"].fillna(0) * 0.5 + df["amount_ratio"].fillna(0) * 100 * 0.5

    return df.sort_values("score", ascending=False).head(top_n)


def add_stock_board_tags(stock_df):
    conn = psycopg2.connect(DATABASE_DSN)

    sql = """
    SELECT code, board_type, board_name
    FROM stock_board_map
    """

    mp = pd.read_sql(sql, conn)
    conn.close()

    industry_map = (
        mp[mp["board_type"] == "行业"]
        .groupby("code")["board_name"]
        .apply(lambda x: list(set(x)))
        .to_dict()
    )

    concept_map = (
        mp[mp["board_type"] == "概念"]
        .groupby("code")["board_name"]
        .apply(lambda x: list(set(x)))
        .to_dict()
    )

    df = stock_df.copy()
    df["industry_tags"] = df["code"].map(industry_map)
    df["concept_tags"] = df["code"].map(concept_map)

    df["industry_tags"] = df["industry_tags"].apply(lambda x: x if isinstance(x, list) else [])
    df["concept_tags"] = df["concept_tags"].apply(lambda x: x if isinstance(x, list) else [])

    return df
```

然后筛选：

```
def select_board_linkage_real(stock_df, top_n=5):
    hot_boards = get_hot_boards(top_n=30)
    hot_board_names = set(hot_boards["board_name"].tolist())

    df = add_stock_board_tags(stock_df)

    def calc_board_hit(row):
        tags = set(row["industry_tags"] + row["concept_tags"])
        hit = tags & hot_board_names
        return list(hit)

    df["hot_board_hits"] = df.apply(calc_board_hit, axis=1)
    df["hot_board_hit_count"] = df["hot_board_hits"].apply(len)

    cond = (
        (df["hot_board_hit_count"] >= 1)
        & (df["pct_chg"] >= 3)
        & (df["volume_ratio"] >= 1.2)
        & (df["amount"] >= 200000000)
        & (~df["name"].str.contains("ST|退", na=False))
    )

    result = df[cond].copy()

    result["linkage_score"] = (
        result["hot_board_hit_count"] * 20
        + result["pct_chg"].fillna(0) * 2
        + result["volume_ratio"].fillna(0) * 5
        + result["turnover"].fillna(0)
    )

    result = result.sort_values("linkage_score", ascending=False).head(top_n)

    return result
```

# 四、你完全没行业经验的话，怎么理解这两个模块？

你可以用一个很简单的逻辑记住：

## 1. 板块成交占比历史表 = 看钱往哪里去

不是看哪个板块今天涨了，而是看：

```
资金是不是连续几天越来越集中？
```

这比单日涨幅更重要。

比如：

```
A板块今天涨5%，但成交占比从8%降到4%
```

这可能是高潮后的兑现。

而：

```
B板块今天只涨1.5%，但成交占比连续5天从3%升到7%
```

这可能是资金正在悄悄建仓。

所以这个模块判断的是：

```
主线正在形成，还是已经退潮？
```

------

## 2. 个股板块映射 = 判断股票是不是主线里的票

一只股票上涨，有两种情况：

```
孤立上涨：只有它自己涨
板块联动：它所属的行业/概念也在涨
```

交易员更重视后者，因为板块联动意味着：

```
不是单一资金乱拉，而是整个方向被市场认可
```

所以板块联动选股的核心是：

```
强势板块 + 板块内强势个股 + 放量 + 技术形态
```

# 六、你应该优先做哪一步？

建议顺序是：

```
第一步：先做 stock_board_map
第二步：再做 board_amount_ratio
第三步：最后改造板块联动选股
```

原因是：

`stock_board_map` 不依赖历史数据，跑一次就能用；
 `board_amount_ratio` 至少要积累 3-5 个交易日后，3日/5日变化才有意义。

所以刚开始几天，系统可以这样降级：

```
历史数据不足 3 天：只显示当日板块强度
历史数据达到 3 天：显示 3日成交占比变化
历史数据达到 5 天：显示 5日成交占比变化
```
