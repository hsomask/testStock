V3：统一同花顺板块口径”技术方案。核心目标是：先不要继续优化 V2 生命周期，先把板块分类体系统一到同花顺；否则趋势追踪越优化，底层口径越乱。

# V3：统一同花顺板块分类口径

## 背景

当前系统已经具备：

- 每日复盘报告；
- 观察池；
- 交易计划；
- 同花顺题材归因；
- board_history 板块成交占比；
- board_trend_tracker 板块资金趋势追踪 V2。

但目前板块分类口径存在不一致问题：

1. 报告中存在行业层级重复，例如：
   - 白酒Ⅱ / 白酒Ⅲ；
   - 证券Ⅱ / 证券Ⅲ；
   - 体育Ⅱ / 体育Ⅲ；
   - 电子化学品Ⅱ / 电子化学品Ⅲ。

2. 用户实际看盘主要使用同花顺，但报告中的行业、概念、板块名称与同花顺看盘界面不完全一致。

3. 当前 `stock_board_map`、`board_history`、`board_trend_tracker` 可能混用了：
   - 东方财富板块；
   - 申万行业；
   - 同花顺概念；
   - 自建映射；
   - 动态概念板块。

这会导致：

```text
报告里的板块 ≠ 用户在同花顺里看到的板块

因此，V3 的优先级是：

先统一板块分类口径，再继续优化趋势追踪 V2。
一、V3 目标

统一系统中的板块分类展示和趋势追踪口径，默认使用：

同花顺行业板块
同花顺概念板块
同花顺强势股题材归因

具体目标：

新增同花顺行业 / 概念成分股映射表；
board_history 使用同花顺映射聚合板块成交额；
board_amount_ratio 增加 source 字段，区分 ths / eastmoney / sw；
board_trend_tracker 默认只读取同花顺口径数据；
日报行业 / 概念展示优先使用同花顺名称；
旧口径保留为 fallback，不直接删除；
不改变选股策略和交易计划逻辑。
二、整体原则
1. 不直接删除旧数据

不要直接清空或删除现有：

stock_board_map
board_amount_ratio

而是新增 source 字段或新增同花顺专用表。

推荐做法：

保留旧口径；
新增 ths 口径；
报告默认使用 ths。
2. 板块分类统一，不等于行情源全部切换

本次只统一：

板块名称
板块分类
个股归属
板块成交额聚合口径

个股行情仍然可以继续使用当前稳定的数据源：

东方财富 / AkShare / Sina / Tencent fallback

也就是说：

行情数据源可以继续混合；
板块分类展示口径统一为同花顺。
3. 不修改交易计划

本次不修改：

trade_plan.py
selector.py
risk 规则
仓位规则
候选低吸规则

V3 只是让“板块和趋势分析”跟同花顺看盘口径一致。

三、数据库设计

有两个可选方案。

方案 A：扩展原表，推荐

在现有 stock_board_map 增加字段：

ALTER TABLE stock_board_map
ADD COLUMN IF NOT EXISTS source VARCHAR(30) DEFAULT 'unknown';

ALTER TABLE stock_board_map
ADD COLUMN IF NOT EXISTS board_level VARCHAR(30);

ALTER TABLE stock_board_map
ADD COLUMN IF NOT EXISTS updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP;

含义：

source:
  ths         同花顺
  eastmoney   东方财富
  sw          申万
  unknown     历史旧数据

board_level:
  industry    行业
  concept     概念
  dynamic     动态概念

写入同花顺数据时：

source = 'ths'
board_type = '行业' / '概念'
board_level = 'industry' / 'concept'
方案 B：新建同花顺专用表

如果不想动原表，可以新建：

CREATE TABLE IF NOT EXISTS stock_board_map_ths (
    id SERIAL PRIMARY KEY,
    code VARCHAR(20) NOT NULL,
    name VARCHAR(100),
    board_type VARCHAR(20) NOT NULL,
    board_code VARCHAR(50),
    board_name VARCHAR(100) NOT NULL,
    board_level VARCHAR(30),
    source VARCHAR(30) DEFAULT 'ths',
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE (code, board_type, board_name)
);

推荐方案 A，因为后续 board_history.py 只需要加 WHERE source='ths'，改动更小。

board_amount_ratio 增加 source 字段

为了避免旧口径和同花顺口径混在一起，给 board_amount_ratio 增加：

ALTER TABLE board_amount_ratio

建议唯一约束改成：

如果原来已经有唯一约束，需要检查 schema 后再改，避免冲突。

七、写入数据库逻辑

然后全量插入。

插入语句：

INSERT INTO stock_board_map (
    code, name, board_type, board_code, board_name,
    board_level, source, updated_at
)
VALUES (%s, %s, %s, %s, %s, %s, 'ths', CURRENT_TIMESTAMP)
ON CONFLICT (code, board_type, board_name, source)
DO UPDATE SET
    name = EXCLUDED.name,
    board_code = EXCLUDED.board_code,
    board_level = EXCLUDED.board_level,
    updated_at = CURRENT_TIMESTAMP;

十五、风险点
1. AkShare 同花顺接口可能慢

同花顺概念板块很多，逐个拉成分股可能比较慢。

处理方式：

ths_board_mapper 不放进每天主流程；
先手动跑；
后续周末跑。
2. 部分板块成分股为空

接口可能返回空表或失败。

处理方式：

记录 failed_boards；
跳过该板块；
不中断全流程。
3. 同花顺板块和东方财富涨幅榜可能不一致

如果 fetch_industry_boards() / fetch_concept_boards() 仍然用东财，而趋势追踪用同花顺，日报仍可能存在口径差异。

第一版允许：

趋势追踪用同花顺口径；
行情榜单仍用现有数据。

但后续 V3.1 可继续把行业 / 概念涨幅榜也切到同花顺。

十七、当前优先级

先不要继续调 V2 生命周期阈值。

当前优先顺序：

1. 建立同花顺板块映射；
2. 用同花顺映射生成板块成交占比；
3. 趋势追踪默认读取同花顺口径；
4. 日报标注同花顺口径；
5. 重新积累 5-10 天数据；
6. 再优化生命周期判断。
十八、最终目标

用户平时使用同花顺看盘，因此报告中的板块名称、板块趋势、板块资金流向要能直接对应同花顺界面。

最终效果：

报告说“先进封装资金流入增强”，用户能直接在同花顺找到先进封装板块；
报告说“存储芯片进入分歧期”，用户能直接在同花顺验证板块走势；
报告说“半导体概念成交占比提升”，用户能直接在同花顺对照盘面。

这样系统才真正从“能生成报告”升级为“能辅助实际看盘”。
