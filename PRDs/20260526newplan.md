def _classify_lifecycle(row):
    """
    根据资金状态、趋势评分、成交占比变化、涨幅、上涨比例判断板块生命周期。
    """

建议逻辑：

主升期

条件：

trend_score >= 70
and amount_ratio_change_10d > 0
and amount_ratio_change_5d > 0
and up_ratio >= 0.55
and latest_pct_chg > 0

含义：

中短期资金持续流入，板块上涨有扩散，主线确认度较高。
加速期

条件：

flow_status == "加速流入"
or (
    amount_ratio_change_5d > 0.002
    and amount_vs_5d_mean >= 1.3
    and latest_pct_chg > 1
)

含义：

短期成交占比快速提升，资金明显加速进入。
启动期

条件：

amount_ratio_change_5d > 0
and trend_score >= 40
and trend_score < 70
and latest_pct_chg > 0

含义：

板块刚出现资金关注，但持续性还需要确认。
分歧期

条件：

flow_status == "高位分歧"
or (
    latest_amount_ratio >= amount_ratio_mean_20d
    and amount_vs_5d_mean >= 1.2
    and (latest_pct_chg <= 1 or up_ratio < 0.5)
)

含义：

成交活跃但上涨扩散不足，板块内部开始分歧。
修复期

条件：

flow_status == "资金回流"
or (
    amount_ratio_change_5d > 0
    and amount_ratio_change_10d <= 0
    and latest_pct_chg > 0
)

含义：

前期资金走弱后，近期出现回流修复。
退潮期

条件：

flow_status == "资金退潮"
or (
    amount_ratio_change_5d < 0
    and latest_pct_chg < 0
)

含义：

成交占比下降，价格转弱，资金热度下降。
沉寂期

其他情况：

无明显资金方向，暂时不是重点。
3. 生命周期优先级

判断顺序很重要，建议按下面顺序：

分歧期
退潮期
加速期
主升期
修复期
启动期
沉寂期

理由：

分歧和退潮是风险信号，优先识别；
加速期是短线强信号；
主升期是持续性信号；
修复期和启动期次之；
其他归为沉寂期。

3. 新增迁移判断函数
def _classify_lifecycle_signal(prev_stage, curr_stage):
    if not prev_stage:
        return "新增观察"

    positive = {
        ("沉寂期", "启动期"): "新启动",
        ("启动期", "加速期"): "主线加强",
        ("启动期", "主升期"): "主线确认",
        ("修复期", "启动期"): "修复转强",
        ("修复期", "加速期"): "资金回流加强",
        ("加速期", "主升期"): "主线延续",
        ("主升期", "主升期"): "主线延续",
    }

    warning = {
        ("主升期", "分歧期"): "主线分歧",
        ("加速期", "分歧期"): "加速后分歧",
        ("分歧期", "退潮期"): "分歧转退潮",
        ("主升期", "退潮期"): "主线退潮",
        ("启动期", "退潮期"): "启动失败",
        ("修复期", "退潮期"): "修复失败",
    }

    if (prev_stage, curr_stage) in positive:
        return positive[(prev_stage, curr_stage)]

    if (prev_stage, curr_stage) in warning:
        return warning[(prev_stage, curr_stage)]

    if curr_stage == prev_stage:
        return "状态延续"

    if curr_stage in ["启动期", "加速期", "主升期"]:
        return "状态转强"

    if curr_stage in ["分歧期", "退潮期"]:
        return "状态转弱"

    return "状态变化"
六、主线连续天数

新增字段：

mainline_streak

定义：

如果一个板块最近连续处于以下阶段：

启动期 / 加速期 / 主升期 / 修复期

则记为主线/关注状态。

连续天数计算：

def _calc_mainline_streak(history_stage_list):
    positive_stages = {"启动期", "加速期", "主升期", "修复期"}
    streak = 0
    for stage in reversed(history_stage_list):
        if stage in positive_stages:
            streak += 1
        else:
            break
    return streak

第一版如果实现复杂，可以先不做逐日生命周期历史，而是简单用：

amount_ratio_change_5d > 0 且 pct_positive_days_5d >= 3

近似为：

mainline_streak = min(amount_ratio_up_streak + pct_positive_days_5d, 5)

但更推荐真实计算每日生命周期。
